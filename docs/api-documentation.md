# API Documentation

Full interactive OpenAPI docs are auto-generated by FastAPI and served at
`http://localhost:8000/docs` (Swagger UI) and `/redoc` whenever the backend is running.
This document gives worked examples of the main flows using real responses captured against
the generated dataset.

Base URL: `http://localhost:8000/api/v1`

## Error format

Every error follows the same shape (see `app/core/exceptions.py`):

```json
{
  "error": {
    "code": "ENTITY_NOT_FOUND",
    "message": "Account ACC-DOES-NOT-EXIST was not found.",
    "details": {},
    "request_id": "2b208685-3af6-4ffe-bd12-3cdbc9d98376"
  }
}
```

`code` is one of: `ENTITY_NOT_FOUND` (404), `INVALID_INPUT` / `VALIDATION_ERROR` /
`INVALID_DATE_RANGE` (422), `UNSUPPORTED_FRAUD_RULE` (400), `GRAPH_UNAVAILABLE` (503),
`GRAPH_QUERY_ERROR` / `GRAPH_PROJECTION_ERROR` (502), `INTERNAL_ERROR` (500).

## Running fraud detection

```bash
curl -X POST http://localhost:8000/api/v1/fraud/run-detection
```

```json
{
  "duration_seconds": 13.11,
  "signals_per_rule": {
    "run_fd001_shared_device": 150,
    "run_fd002_shared_ip": 2706,
    "run_fd003_circular_transfer": 17,
    "...": "..."
  },
  "total_signals": 5786,
  "entities_scored": 3485,
  "alerts_created": 734
}
```

Run a single rule (e.g. after tuning a threshold):

```bash
curl -X POST http://localhost:8000/api/v1/fraud/run-rule/FD-003
# {"rule_id":"FD-003","signals_found":17,"alerts_created":0}
```

Unsupported rule id returns a typed 400:

```bash
curl -X POST http://localhost:8000/api/v1/fraud/run-rule/FD-999
```

```json
{
  "error": {
    "code": "UNSUPPORTED_FRAUD_RULE",
    "message": "Unknown or unsupported fraud rule: FD-999",
    "details": {"supported_rules": ["FD-001", "FD-002", "..."]},
    "request_id": "..."
  }
}
```

## Investigation case lifecycle

```bash
curl -X POST http://localhost:8000/api/v1/investigations \
  -H "Content-Type: application/json" \
  -d '{"title": "Ring around ACC-003667", "description": "Shared device + community", "account_ids": ["ACC-003667"]}'
```

```json
{
  "case_id": "CASE-0FB97FF52B",
  "title": "Ring around ACC-003667",
  "status": "OPEN",
  "priority": "MEDIUM",
  "assigned_to": "unassigned",
  "...": "..."
}
```

```bash
curl -X PATCH http://localhost:8000/api/v1/investigations/CASE-0FB97FF52B \
  -H "Content-Type: application/json" -d '{"status": "IN_REVIEW"}'
```

Invalid status values are rejected (422) with the valid set in `details`:

```json
{
  "error": {
    "code": "INVALID_INPUT",
    "message": "Invalid case status: NOT_A_STATUS",
    "details": {"valid_statuses": ["ESCALATED", "FALSE_POSITIVE", "IN_REVIEW", "OPEN", "RESOLVED"]},
    "request_id": "..."
  }
}
```
